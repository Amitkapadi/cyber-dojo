overview of how cyber-dojo's language docker-containers work

cyber-dojo probes the host server to see if docker is installed. If it is then when you press the home page's [create] button cyber-dojo will only offer entries whose languages/ subfolder's manifest.json file has an image_name entry that exists. For example, if cyberdojo/languages/Java-1.8_JUnit/manifest.json contains this...
{ ... "display_name": "Java, JUnit", "image_name": "cyberdojo/java-1.8_junit" }
then Java, JUnit will only be offered if the docker image cyberdojo/java-1.8_junit exists on the host server, as determined by running
$ docker images


cyber-dojo will re-start the docker image_name container to execute an animals cyber-dojo.sh file each time the animal presses the [test] button. See lib/DockerTestRunner.rb
